    NB-IoT server doc and tutorials

    Welcome! This is the documentation site for the Telenor NB-IoT service. Please join our Slack channel if you have any questions.

    Getting started

    Start by going through the interactive terminal tutorial, then Getting started with Telenor NB-IoT Developer Platform to get a quick feel for how the devices and console works.

    If you want to implement a service that receives data you can use one of our client libraries. If your language is not yet supported, then have a look at the documentation for outputs which shows you how the data is sent to e.g. MQTT.

    Backend IP address and APN

    This is the official address for the NB-IoT backend:

    Send data to 172.16.15.14 on port 1234

    The APN on your device must be set to mda.ee if you want to send data to the IoT backend. Other APNs does not work.

    SARA N2 AT command cheat sheet

    Send UDP packet on device

    Create socket (NSOCR), send package (NSOST) and close socket (NSOCL)

    AT+NSOCR="DGRAM",17,8888,1
AT+NSOST=0,"172.16.15.14",1234,12,"48656C6C6F20576F726C6421"
AT+NSOCL=0

    Check IMSI and IMEI

    AT+CIMI
AT+CGSN=1

    Check IP address

    If the module has an assigned IP address it is online and ready to use

    AT+CGPADDR

    Check APN setup

    It should say “mda.ee” if the correct APN is selected

    AT+CGDCONT?

    Ping an external host

    AT+NPING="172.16.0.100"

    It should say +NPING: "172.16.0.100",<number>,<number> if successful.

    Set up for EE APN

    Turn off autoconnect for the module, then reboot

    AT+NCONFIG="AUTOCONNECT","FALSE"
AT+NRB

    Set the MDA/APN name

    AT+CGDCONT=0,"IP","mda.ee"

    Set autoconnect on and reboot

    AT+NCONFIG="AUTOCONNECT","TRUE"
AT+NRB

    Get IMSI and IMEI

    AT+CIMI
AT+CGSN=1

    Register device in Horde, then reboot

    AT+NRB

    Verify that the MDA/APN is set and that you get an IP address

    AT+CGDCONT?
AT+CGPADDR
